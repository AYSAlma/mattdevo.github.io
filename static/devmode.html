Developer Mode Basics

Putting your ChromeOS device into developer mode relaxes some of the restrictions in the standard / verified boot mode, and gives the user a bit more control over the system:

  • Provides full root access to the system via a terminal shell
  • Allows installation of Linux inside ChromeOS via crouton
  • Allows for booting other operating systems via Legacy Boot Mode (aka SeaBIOS) on supported systems
  • Allows for installation of custom firmware components (some of which require removing the firmware write-protect)

Because developer mode is inherently less secure than standard / verified boot mode, when powering on you will be greeted by the developer mode boot screen:

The dev mode boot screen will show for 30 seconds, then beep and boot into the default OS. You can bypass this timeout by pressing [CTRL][D] to boot into ChromeOS, or [CTRL][L] to boot in legacy mode (SeaBIOS) if enabled. The length of the timeout and the default OS (ChromeOS or legacy) can be changed by setting the firmware boot flags (aka GBB flags).

Enabling Developer Mode

On Most ChromeOS devices, entering developer mode requires you to first boot into recovery mode. For Chromebooks, this means pressing [ESC][Refresh][Power]; On Chromeboxes, there is usually a physical recovery button (often a small hole above the SD card reader) which must be depressed when powering on. Once at the recovery screen, press [CTRL][D] to enable developer mode, then confirm when prompted. As a security measure, transitioning to/from developer mode will wipe out all ChromeOS user data, essentially resetting the device.

Exiting developer mode is as simple as following the instructions on the developer mode boot screen (usually pressing [SPACE]), but may require resetting the firmware boot flags if you've changed them. As with entering developer mode, exiting will wipe all ChromeOS user data, so if you plan on keeping your device in developer mode, it's a good idea to set the firmware boot flags to prevent accidental exiting and loss of data.

Getting a Root Shell

Once in developer mode, there are two ways to get a root shell, which is required for doing anything useful:

  1. On the login screen, press [CTRL][ALT][F2] ([=>] on ChromeOS keyboards), then login with user 'chronos' (no password is required, nor should one be set)
  2. When logged in (as a guest is ok) and with a browser window open, press [CTRL][ALT][T], then type 'shell' and press enter
Once you have a root shell, you can do things like install Linux (via crouton or chrx), update the firmware, etc.

Developer Mode Boot Parameters

Several boot-related parameters are stored in non-volatile storage (nvram); these can be modified from a root shell using the crossystem command. As the nvram storage is battery backed, these settings can be reset (either accidentially or on purpose) by disconnecting or completely draining the battery (on Chromebooks) or disconnecting the CMOS battery (on Chromeboxes). These parameters can be overriden with the firmware boot flags (GBB flags), though that requires disabling the firmware write protection. Some useful boot parameters include:

  • Legacy boot mode (enable/disable):
    • # sudo crossystem dev_boot_legacy=[1/0]
  • External media boot (enable/disable):
    • # sudo crossystem dev_boot_usb=[1/0]
    • NOTE: this only applies to booting ChromeOS/ChromiumOS from external media; it has nothing to do with booting Linux/Windows etc, which is done via Legacy Boot Mode
  • Default boot mode/target:
    • # sudo crossystem dev_default_boot=[disk/usb/legacy]
    • NOTE: as above, USB here refers to ChromeOS/ChromiumOS on USB - not Linux

Most users won't need to set these boot parameters manually, since they are set automatically by the ChromeOS Firmware Utility script, but it's useful to know what they are and what they do regardless.
